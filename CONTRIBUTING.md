# Как добавить или изменить урок

Спасибо за вклад в проект. Контент отделён от кода, поэтому добавить урок можно, не зная React.

## Шаг за шагом: новый урок

1. Найдите нужный модуль в `content/`, например `content/level-1-osnovaniya/02-pervye-shagi/`.
2. Создайте файл урока: `NN-slug.md`, где `NN` — порядковый номер (`03`, `04`…), а `slug` — короткое имя латиницей без пробелов. Начните файл с заголовка первого уровня:

   ```markdown
   # Название урока

   Текст урока…
   ```

3. Создайте тест рядом: `NN-slug.quiz.json` с тем же `NN-slug`. Формат — ниже.
4. Проверьте локально (`npm run dev`) и оформите pull request.

Чтобы добавить **модуль**, создайте папку `NN-имя-модуля/` с файлом `_meta.json`:

```json
{ "title": "Название модуля", "description": "Краткое описание" }
```

Для **уровня** — то же самое плюс поле `audience`:

```json
{ "title": "Название уровня", "audience": "Для кого", "description": "Описание" }
```

## Формат теста (`.quiz.json`)

```json
{
  "passing_score": 80,
  "questions": [
    {
      "type": "single",
      "q": "Текст вопроса с одним правильным ответом",
      "options": ["Вариант 1", "Вариант 2", "Вариант 3"],
      "answer": 1,
      "explanation": "Показывается после ответа (необязательно)."
    },
    {
      "type": "multiple",
      "q": "Вопрос с несколькими правильными ответами",
      "options": ["A", "B", "C", "D"],
      "answer": [0, 2]
    },
    {
      "type": "boolean",
      "q": "Утверждение, которое нужно оценить как верное или неверное",
      "answer": true
    }
  ]
}
```

Поля:

- `passing_score` — проходной балл в процентах (обычно 80).
- `type` — `single` (один ответ), `multiple` (несколько), `boolean` (верно/неверно).
- `answer` — индекс варианта (`single`), массив индексов (`multiple`) или `true`/`false` (`boolean`). Индексы считаются с нуля.
- `options` — варианты ответа. Для `boolean` можно не указывать (по умолчанию «Верно» / «Неверно»).
- `explanation` — необязательное пояснение.

## Оформление цитат Писания

Используйте Markdown-цитату с тегом `<cite>` для ссылки:

```markdown
> Текст стиха.
> <cite>Иоанна 3:16</cite>
```

Используйте перевод в общественном достоянии (например, Синодальный) либо тот, на использование которого у проекта есть право.

## Доктринальное рецензирование

Поскольку речь о вероучении, материалы с доктринальным содержанием проходят проверку доверенными редакторами (служителями) перед публикацией. Рекомендуется включить защиту ветки `main` и обязательное ревью pull request.

Подход проекта к спорным внутри евангельского движения вопросам — **обзорный**: представляем разные взгляды и отсылаем читателя к учению его собственной церкви, а не навязываем одну позицию. Образец — урок о крещении.
