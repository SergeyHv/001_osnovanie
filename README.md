# Основание

Открытая платформа последовательного обучения для евангельских христиан — от первых шагов веры до подготовки к служению. Каждый урок завершается тестом; следующий урок открывается только после того, как тест сдан.

Сайт статический (React + TypeScript + Vite), без бэкенда: прогресс ученика хранится в браузере (localStorage). Уроки и тесты отделены от кода и лежат обычными файлами в папке `content/`, поэтому материалы можно добавлять и рецензировать через pull request, не трогая код.

## Запуск локально

Нужен Node.js 18+ (рекомендуется 20).

```bash
npm install      # установить зависимости
npm run dev      # запустить локально (http://localhost:5173)
npm run build    # собрать продакшен-версию в dist/
npm run preview  # посмотреть собранную версию локально
```

## Публикация на GitHub Pages

1. Создайте репозиторий и запушьте в него этот код в ветку `main`.
2. В настройках репозитория откройте **Settings → Pages** и в разделе **Build and deployment** выберите источник **GitHub Actions**.
3. При каждом пуше в `main` workflow `.github/workflows/deploy.yml` соберёт и опубликует сайт автоматически.

Важно про адрес сайта. По умолчанию в `vite.config.ts` стоит `base: '/osnovanie/'` — это подходит, если сайт публикуется по адресу `https://<логин>.github.io/osnovanie/`. Если имя репозитория другое, поменяйте `base` на `/<имя-репозитория>/`. Если используете собственный домен или репозиторий вида `<логин>.github.io`, поставьте `base: '/'`.

Маршрутизация сделана через `HashRouter` (адреса с `#`), поэтому ссылки на уроки не дают ошибку 404 при обновлении страницы на GitHub Pages.

## Как устроен контент

```
content/
└── level-1-osnovaniya/            # уровень (папка с числовым префиксом задаёт порядок)
    ├── _meta.json                 # название, аудитория, описание уровня
    └── 01-evangelie/              # модуль
        ├── _meta.json             # название и описание модуля
        ├── 01-kto-takoy-iisus.md         # текст урока (Markdown)
        └── 01-kto-takoy-iisus.quiz.json  # тест к этому уроку
```

Правила:

- Порядок уровней, модулей и уроков задаётся числовым префиксом в имени папки/файла (`01-`, `02-`…).
- Заголовок урока берётся из первого `# Заголовок` в Markdown.
- Тест — это файл с тем же именем, что и урок, плюс `.quiz.json`. Если файла нет, урок считается пройденным сразу после открытия.
- Цитаты Писания удобно оформлять как цитату Markdown с `<cite>` для ссылки (см. готовые уроки как образец).

Подробности о добавлении уроков — в [CONTRIBUTING.md](./CONTRIBUTING.md).

## Уровни сложности (план)

1. **Основания** — для новообращённых.
2. **Ученичество** — рост и базовые духовные навыки.
3. **Углублённое изучение** — систематическое богословие, обзор книг Библии.
4. **Толкование и навыки** — герменевтика, апологетика, история церкви.
5. **Служение** — гомилетика, душепопечение, наставничество, пасторская этика.

В стартовый набор включены полностью рабочие уроки Уровня 1 и один урок Уровня 2 как образцы.

## Подход к спорным вопросам

Проект подаёт спорные внутри евангельского движения темы (крещение, эсхатология, действие даров Духа) **обзорно**, представляя разные взгляды, а не навязывая одну позицию. Урок о крещении — образец такого подхода.
